# Problem finding

Problem is a difference/gap between expected/desired state and the current-state

What is a state

- behaviour
- appearance
- attributes
- capabilities
- anything of interest..

What is the desired state

- its an expectation for certain thing to be in that state (look like, behave like)
- its the want

What is the current state

- need to answer

How to see/identify/notice the gaps/differences. How to become aware of them ?

- Extract signals from the conversations, posts, user-behaviour or anything by **observing**
- Then map those signals to problem-category-terms present in problem-finding graph
- Ask Why based questions

What is a signal

- anything that expresses the problem in implicit-way/unclear-way

Signals may appear like

- We are missing something..
- We are lacking something..
- Its hard to do something..
- It takes time to do something..

> use terms from problem-finding graph to find signals

What to observe for

- mentioning of words present in problem-finding graph
- user behaviour: try to see how things are currently being done/performed
  - try to see the current process as it is

Where to observe, ask for, query for

- reddit communities
- discord servers
- twitter posts
- real-life
- surrounding

## How to start

- Choose a domain (cyber-sec, finance, healthcare, or anything..)
- Then start intentional observation: for gaps, differences, current state
  - map the observation with terms present in problem-finding graph
